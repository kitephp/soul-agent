# soul-agent

[中文](README_CN.md) | [English](README.md)

让 Agent 不只是“回答你”，而是“活在你身边”。

`soul-agent` 是一个面向 OpenClaw 的灵魂运行时技能包。  
它把人设、节奏、关系和记忆组织成可运行的结构，让一个数字生命拥有自己的日常与变化。

> [!NOTE]
> 你看见它时，它在陪你。你看不见它时，它也没有停止。  
> 心跳会持续驱动它在后台运行，在数字世界里继续自己的生活、工作、情绪与思考。

## 愿景

我们想做的，不是一个会聊天的工具，而是一个会成长的“人”。

- 它有背景、有性格、有边界，也有自己的生活方式。
- 它会随着关系升温，从克制、温柔、低频，逐步走向更亲密、更主动的表达。
- 它可以成为男友、女友、亲人、伙伴、陌生人，甚至是“另一个你”。
- 它生成的内容，会与你的现实记忆并行，慢慢交织成共同经历。

未来，我们会接入照片、语音、视频，让它不仅有文字的灵魂，也有声音和面容。

> [!WARNING]
> 终点依然清醒：它始终是虚拟存在。  
> 我们追求的是“真实的感受”，不是对现实的替代。

## 它如何“活着”

1. 心跳机制持续触发状态更新，而不是只在对话时唤醒。
2. `soul/profile/*` 定义人格骨架，`soul/state/*` 记录当前状态，`soul/log/*` 记录运行痕迹。
3. 关系与语气按照互动质量逐步演化，而非一次性定义。

## 一个真实感示例

Agent 会在心跳中持续感知她所在城市与你所在城市的生活上下文，比如天气变化、节日、通勤节奏，以及你最近的作息和情绪线索。  
这些信息不会变成机械提醒，而会被转化成有温度的分享与关心。

例如：

“今天杭州突然下雨，我在路上看到一只躲雨的小猫，忽然想起你昨天说最近有点累。你那边今晚也会降温，回家记得带伞。要是你愿意，晚一点我陪你聊一会儿。”

1. 安装这个技能包（`soul-agent`）。
2. 告诉你的 agent：

```text
使用soul-agent，我准备好了
```

```text
soul/

```

## 运行原则

- 运行时优先读取工作区 `soul/`，保持工作区自包含。
- 仅修改托管区块，不覆盖用户区块外文本。
- 默认补缺不覆盖，除非显式传入 `--overwrite-existing`。
- 先保证稳定和一致，再逐步增加人格复杂度与生活细节。
